# Autonomous Maze Navigation - ROS 2 Project

This repository contains a ROS 2 implementation of an autonomous navigation system for a mobile robot operating within a simulated maze environment. The core of the navigation logic is based on the **Artificial Potential Field (APF)** method, integrated with **ROS 2 Jazzy** and **Gazebo Harmonic**.

## Core Concept

The navigation system treats the robot as a particle moving under the influence of a virtual potential field. The field is composed of two primary components:
- **Attractive Potential:** A force that pulls the robot toward the target coordinates.
- **Repulsive Potential:** A force generated by obstacles (detected via LiDAR) that pushes the robot away to prevent collisions.

By combining these vectors, the robot determines its optimal heading and velocity in real-time.

## Package Structure

- `maze_navigation/`: Main Python package containing the planner node.
- `launch/`: Contains `maze_sim.launch.py` to initialize the simulation environment.
- `worlds/`: Includes `simple_maze.world` for the Gazebo simulation.
- `resource/` & `package.xml`: Standard ROS 2 package metadata and resources.

## Installation & Setup

### 1. Environment Preparation
Ensure you have ROS 2 Jazzy and Gazebo Harmonic installed. Then, install the necessary bridge and SDK packages:
```bash
sudo apt update
sudo apt install -y ros-jazzy-ros-gz ros-jazzy-dynamixel-sdk
```

### 2. Workspace Initialization
Create a new workspace and clone this repository along with the required TurtleBot3 dependencies:
```bash
mkdir -p ~/ros2_project_ws/src
cd ~/ros2_project_ws/src

# Clone this project
git clone https://github.com/Ahmedosrf/ros2-maze-navigation.git maze_navigation

# Clone TurtleBot3 dependencies (Jazzy branch)
git clone -b jazzy https://github.com/ROBOTIS-GIT/turtlebot3_msgs.git
git clone -b jazzy https://github.com/ROBOTIS-GIT/turtlebot3.git
git clone -b jazzy https://github.com/ROBOTIS-GIT/turtlebot3_simulations.git
```

### 3. Build the Project
```bash
cd ~/ros2_project_ws
source /opt/ros/jazzy/setup.bash
rosdep install --from-paths src --ignore-src -r -y
colcon build --symlink-install
```

## Execution Guide

Follow these steps in separate terminals to run the simulation and the navigation planner.

### Step 1: Launch Simulation
Start the Gazebo environment and spawn the robot:
```bash
source ~/ros2_project_ws/install/setup.bash
export TURTLEBOT3_MODEL=burger
ros2 launch maze_navigation maze_sim.launch.py
```

### Step 2: Run Navigation Planner
In a new terminal, execute the Potential Field planner node with your desired goal coordinates:
```bash
source ~/ros2_project_ws/install/setup.bash
ros2 run maze_navigation potential_field_planner --ros-args -p goal_x:=9.0 -p goal_y:=9.0
```

## Implementation Details

- **LiDAR Filtering:** The system automatically handles invalid sensor data (NaN, Inf, or 0.0) by substituting them with the maximum sensor range, ensuring stable force calculations.
- **Dynamic Tuning:** Parameters such as `k_att` (attractive gain), `k_rep` (repulsive gain), and `d_obs` (influence distance) can be tuned via ROS parameters to optimize performance for different maze layouts.
- **Precision Stopping:** The robot is programmed to cease all movement once it enters a 0.2-meter radius of the target goal.
